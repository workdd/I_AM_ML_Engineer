# MCP의 다음은 클로드 스킬(Claude Skills)인가?

- **출처**: [요즘IT](https://yozm.wishket.com/magazine/detail/3508/)
- **저자**: 스벨트전도사
- **읽은 날짜**: 2025-12-18
- **태그**: #MCP #ClaudeSkills #LLM #자동화 #Anthropic

## 핵심 내용

2024년 하반기 인기를 끌었던 MCP(Model Context Protocol)의 근본적 문제점을 지적하고, Anthropic의 Claude Skills를 더 나은 대안으로 제시.

### MCP란?
- LLM과 외부 도구를 연결하는 표준화된 규격
- JSON-RPC 스펙, 엔드포인트 설정 필요
- 개발자들이 복잡한 설정 작업을 해야 함

### MCP의 근본적 문제
> "LLM은 자연어 처리에 특화되어 있는데, 왜 정형화된 프로토콜이 필요한가?"

### Claude Skills의 장점
- `SKILL.md` 파일에 **자연어로** 워크플로 정의
- 폴더 기반의 단순한 구조
- 비개발자도 쉽게 작성 가능
- LLM의 본질(자연어)에 맞는 접근

## 인상 깊은 부분

> 반복되는 고정 작업은 스크립트로 분리하여 토큰 소모 감소 및 성능 향상

**사용 시나리오별 추천:**
| 시나리오 | 추천 |
|---------|------|
| 개인 워크플로, 팀 자동화 | Claude Skills |
| 외부 서비스 연동, 대규모 배포 | MCP |

## 내 생각 / 적용점

### 배운 점
- MCP vs Claude Skills는 "정형화 vs 자연어" 접근의 차이
- LLM 특성상 자연어 기반이 더 직관적일 수 있음
- 하지만 대규모/외부 연동에는 MCP의 표준화가 필요

### 실무 적용
- 개인/팀 내부 자동화: Claude Skills로 빠르게 구축
- 프로덕션 서비스 연동: MCP 유지
- 두 가지를 상황에 맞게 혼용하는 것이 현실적
